# التحديات والمسابقات (Challenges)

نظام اختبارات قصيرة تحفز المستخدم على التعلم بطريقة تفاعلية، مقابل نقاط وشهادات تقدير.

---

## 1. Quiz List — `quiz-list.html`

### الغرض (Purpose)
عرض كل الاختبارات المتاحة مع تمييز "تحدي اليوم"، وإظهار تقدم المستخدم في كل اختبار.

### التخطيط (Layout)
1. Top bar بعنوان القسم.
2. **بطاقة تحدي اليوم** (`.daily-challenge`) في الأعلى بإطار accent مميز.
3. **قائمة الاختبارات** (`.quiz-card` ×N): كل بطاقة تحتوي تصنيف (badge)، عنوان، عدد الأسئلة والنقاط، وشريط تقدم (`.progress-track`) يوضح ما أنجزه المستخدم من قبل.
4. Bottom navigation مشترك.

### المكونات (Components)
| المكون | الوصف |
|---|---|
| `.daily-challenge` | بطاقة بارزة بخلفية كريمية وحد accent، منفصلة بصريًا عن باقي القائمة |
| `.quiz-card` | بطاقة قابلة للضغط (`onclick`) تنقل لـ `quiz-question.html?quiz=<id>` |
| `.progress-fill` | شريط تقدم بعرض ديناميكي (`style="width: X%"`) — حاليًا ثابت كـ placeholder، يُفترض حسابه من بيانات حقيقية |

### الانتقال (Navigation)
`quiz-list.html` → `quiz-question.html?quiz=<id>`

---

## 2. Quiz Question — `quiz-question.html`

### الغرض (Purpose)
عرض سؤال اختيار من متعدد مع تغذية راجعة فورية (صحيح/خاطئ) قبل الانتقال للسؤال التالي.

### التخطيط (Layout)
1. Top bar بزر إغلاق (✕) وعداد "سؤال X من Y".
2. شريط تقدم الأسئلة (`.quiz-progress`، منفصل عن شريط تقدم الاختبار في القائمة).
3. نص السؤال (`h2`).
4. قائمة خيارات (`.quiz-options`) ×4 كأزرار.
5. زر "التالي" (`#nextQuestionBtn`) — معطّل (`disabled`) حتى يختار المستخدم إجابة.

### المكونات (Components)
| المكون | الوصف |
|---|---|
| `.quiz-option[data-correct]` | كل خيار يحمل `data-correct="true/false"` ليحدد الإجابة الصحيحة في الـ JS |
| `.is-selected` / `.is-correct` / `.is-wrong` | فئات CSS تُضاف ديناميكيًا بعد الإجابة لتلوين الخيار المختار والصحيح |

### المنطق (Logic - `quiz.js`)
- عند الضغط على خيار: يُعطّل كل الخيارات، يُلوّن الخيار المختار (صحيح/خطأ)، ويُظهر الإجابة الصحيحة إن كانت مختلفة.
- يُفعّل زر "التالي" فقط بعد الإجابة.
- **ملاحظة Boilerplate**: هذا الملف يعرض سؤالًا واحدًا مكتوبًا مباشرة في HTML كنموذج. في البناء الكامل، يجب تحويل الأسئلة لمصفوفة بيانات (أو جلبها من API) وإعادة عرضها ديناميكيًا حسب رقم السؤال الحالي.

### الانتقال (Navigation)
`quiz-question.html` → `quiz-result.html` (بعد آخر سؤال)

## 3. Quiz Result — `quiz-result.html`

### الغرض (Purpose)
الاحتفال بإنجاز المستخدم، عرض نتيجته ونقاطه، ومنحه شهادة تقدير قابلة للمشاركة.

### التخطيط (Layout)
1. رمز احتفالي (🎉) + "أحسنت!".
2. نص النتيجة (`#scoreText`).
3. بطاقة "النقاط المكتسبة" (`.points-earned`).
4. بطاقة "شهادة تقدير" (`.certificate`) بخلفية متدرجة احتفالية.
5. مجموعة أزرار: مشاركة النتيجة، اختبار آخر، رجوع للرئيسية.

### المكونات (Components)
| المكون | الوصف |
|---|---|
| `.certificate` | بطاقة الشهادة الرقمية، تُظهر فقط عند تجاوز نسبة نجاح معينة (منطق يُضاف لاحقًا) |
| `#shareResultBtn` | يستخدم `navigator.share` أو تنبيه بديل |

### المنطق (Logic - `quiz-result.js`)
- يقرأ إجمالي نقاط المستخدم من `localStorage` ويعرضها.
- `Storage.addPoints(50)` يُستدعى من `quiz.js` عند الانتقال لهذه الشاشة (محاكاة لإضافة نقاط الاختبار المُنجز).

### الانتقال (Navigation)
- `quiz-result.html` → `quiz-list.html` ("اختبار آخر")
- `quiz-result.html` → `../home/home.html` ("رجوع للرئيسية")
