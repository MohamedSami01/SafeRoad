# شاشات تسجيل الدخول (Auth)

شاشتان لإدارة دخول المستخدم: تسجيل دخول لمستخدم حالي، وإنشاء حساب لمستخدم جديد. كلاهما يدعمان أيضًا "الدخول كزائر".

---

## 1. Login — `login.html`

### الغرض (Purpose)
السماح للمستخدم الحالي بتسجيل الدخول، أو الانتقال السريع لإنشاء حساب، أو تخطي التسجيل بالكامل والدخول كزائر للوصول المباشر للمحتوى التوعوي.

### التخطيط (Layout)
ترتيب عمودي بسيط من الأعلى للأسفل:
1. رأس (`.auth__header`): شعار + عنوان "أهلاً بك" + جملة توضيحية.
2. نموذج (`.auth__form`): حقلين (بريد/هاتف، كلمة مرور) + رابط "نسيت كلمة المرور" + زر تسجيل دخول.
3. فاصل "أو" (`.auth__divider`).
4. أزرار تسجيل دخول اجتماعي (جوجل / آبل) بشكل `btn-outline`.
5. زر "الدخول كزائر" بشكل نصي بسيط (`btn-text`) لتقليل الإلحاح البصري عليه مقارنة بالتسجيل.
6. سطر تبديل أسفل الشاشة يقود لـ `signup.html`.

### المكونات (Components)
| المكون | الوصف |
|---|---|
| `#loginForm` | نموذج بحقلين مطلوبين (`required`) |
| `#guestBtn` | يخزّن مستخدم زائر في `localStorage` وينتقل مباشرة للرئيسية |
| `.auth__social` | أزرار وهمية (placeholder) لتسجيل الدخول الاجتماعي — تحتاج تكامل OAuth حقيقي عند الربط بـ backend |

### المنطق (Logic - `auth.js`)
- `loginForm` submit → يمنع إعادة تحميل الصفحة، يحفظ بيانات مستخدم وهمية، ينتقل لـ `home.html`.
- `guestBtn` click → يحفظ `{ isGuest: true }`، ينتقل لـ `home.html`.

### الانتقال (Navigation)
`login.html` → `home.html` (بعد الدخول)
`login.html` → `signup.html` (رابط "إنشاء حساب جديد")

---

## 2. Signup — `signup.html`

### الغرض (Purpose)
تمكين مستخدم جديد من إنشاء حساب لحفظ نقاطه، شهاداته، وسجل بلاغاته بشكل دائم.

### التخطيط (Layout)
نفس بنية `login.html` مع زر رجوع علوي (`.auth__back`) واستبدال الحقلين بثلاثة حقول: الاسم الكامل، البريد/الهاتف، كلمة المرور. لا يحتوي على خيارات تسجيل اجتماعي أو زائر لتوجيه المستخدم لمسار واضح واحد فقط (إكمال التسجيل).

### المكونات (Components)
| المكون | الوصف |
|---|---|
| `#signupForm` | نموذج بثلاثة حقول مطلوبة |
| `.auth__back` | يستدعي `goBack()` من `js/navigation.js` |

### المنطق (Logic - `auth.js`)
- `signupForm` submit → يحفظ بيانات المستخدم الجديد في `localStorage` وينتقل لـ `home.html`.

### الانتقال (Navigation)
`signup.html` → `home.html` (بعد إنشاء الحساب)
`signup.html` → `login.html` (رابط "تسجيل الدخول")

---

## الاعتماديات المشتركة (Shared Dependencies)
- `css/variables.css`, `css/base.css`, `css/components.css`, `./auth.css`
- `js/storage.js` (كلا الشاشتين) + `js/navigation.js` (signup فقط، لزر الرجوع)
- ملاحظة: المصادقة هنا Mock بالكامل عبر `localStorage`. يجب استبدالها باستدعاءات API حقيقية (مثل `fetch('/api/login')`) عند ربط الـ backend.
