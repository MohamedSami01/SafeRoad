# الحساب والإعدادات (Account)

شاشتان لإدارة هوية المستخدم وتقدمه، والتحكم في إعدادات التطبيق.

---

## 1. Profile — `profile.html`

### الغرض (Purpose)
عرض تقدم المستخدم وإنجازاته (نقاط، شارات، شهادات، بلاغات) في مكان واحد لتحفيزه على الاستمرار في استخدام التطبيق.

### التخطيط (Layout)
1. Top bar بعنوان "حسابي" وأيقونة إعدادات (⚙️) تنقل لـ `settings.html`.
2. **رأس البروفايل** (`.profile-header`): صورة رمزية، اسم المستخدم، شارة المستوى الحالي (مثل "واعي")، زر تعديل الملف الشخصي.
3. **بطاقة النقاط** (`.points-summary`): العدد الإجمالي للنقاط + شريط تقدم نحو المستوى التالي.
4. **شبكة الشارات** (`.badges-grid`) 3 أعمدة: شارات مكتسبة وشارات مقفلة (`.is-locked` بشفافية أقل).
5. **قائمة الشهادات** (`#certsList`): كل شهادة في صف مع زر "عرض".
6. **رابط بلاغاتي**: بطاقة واحدة تنقل مباشرة لـ `reports-history.html` (في قسم Reporting) بدل تكرار الشاشة هنا.
7. Bottom navigation مشترك.

### المكونات (Components)
| المكون | الوصف |
|---|---|
| `#profileName` / `#totalPoints` | تُحدّثان ديناميكيًا من `account.js` |
| `.badge-item.is-locked` | شارة لم يحصل عليها المستخدم بعد، تُعرض بشفافية لتحفيزه على إكمالها |
| `#editProfileBtn` | حاليًا يعرض تنبيه placeholder؛ يُفترض ربطه بشاشة تعديل بيانات فعلية عند البناء الكامل |

### المنطق (Logic - `account.js`)
- يقرأ اسم المستخدم وعدد نقاطه من `localStorage` عبر `Storage.get()` ويعرضهما.

### الانتقال (Navigation)
- `profile.html` → `settings.html`
- `profile.html` → `../reporting/reports-history.html`

## 2. Settings — `settings.html`

### الغرض (Purpose)
التحكم في تفضيلات الحساب، الإشعارات، الخصوصية، والوصول لمعلومات عن التطبيق.

### التخطيط (Layout)
ثلاث مجموعات إعدادات (`.settings-group`)، كل واحدة بصفوف (`.settings-row`):
1. **عام**: اللغة (قائمة منسدلة)، تذكير بتحدي اليوم، تنبيهات الطوارئ (مفاتيح تبديل).
2. **الخصوصية**: إظهار/إخفاء اسم المستخدم في القصص والبلاغات.
3. **عن التطبيق**: روابط "من نحن"، "تواصل معنا"، "سياسة الخصوصية وشروط الاستخدام".

أسفل المجموعات: زر "تسجيل الخروج" (outline) وزر نصي أحمر "حذف الحساب".

### المكونات (Components)
| المكون | الوصف |
|---|---|
| `#languageSelect` | قائمة منسدلة لتغيير لغة الواجهة (عربي/إنجليزي) — التبديل الفعلي للغة يحتاج طبقة i18n عند البناء الكامل |
| `#logoutBtn` | يطلب تأكيدًا (`confirm()`) قبل مسح بيانات المستخدم والانتقال لتسجيل الدخول |
| `#deleteAccountBtn` | يطلب تأكيدًا أقوى قبل مسح كل البيانات المحلية بالكامل |

### المنطق (Logic - `settings.js`)
- كل من تسجيل الخروج وحذف الحساب يستخدمان `confirm()` كحماية أساسية ضد الضغط بالخطأ، ثم يُنفذان مسح `localStorage` (جزئي أو كلي) والتنقل المناسب.
- **ملاحظة أمان**: `confirm()` هنا أداة بسيطة للنموذج الأولي. في تطبيق حقيقي، يُفضّل استخدام مربع حوار (Modal) مصمم بنفس هوية التطبيق، وربط حذف الحساب بطلب API فعلي للخادم.

### الانتقال (Navigation)
- `settings.html` → `../auth/login.html` (بعد تسجيل الخروج)
- `settings.html` → `../onboarding/splash.html` (بعد حذف الحساب)
